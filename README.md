# Academic Curriculum Automator (Автоматизация РПД)

Данный набор утилит разработан для автоматизации рутинных процессов при работе с **Рабочими Программами Дисциплин (РПД)** и другими учебными документами в формате Microsoft Word. Набор позволяет массово редактировать таблицы согласования, заменять подписи, вставлять сканы, конвертировать форматы и оптимизировать размер файлов.

---

## 📋 Основные возможности

Проект разделен на несколько независимых инструментов, каждый из которых решает свою задачу:

### 1. Конвертация форматов
*   **`run_doc_converter.py`**: Массово переводит старые файлы `.doc` в современный формат `.docx`. Это необходимо для работы большинства других утилит. (Требует установленный MS Word).
*   **`run_convert_to_pdf.py`**: Быстрая конвертация готовых документов `.docx` в формат `.pdf` с сохранением форматирования.

### 2. Работа с листами согласования
*   **`run_approval_update.py`**: Автоматизирует обновление таблицы "Лист согласования".
    *   Ищет таблицу по ключевым словам (согласовано, учебный год, РПД).
    *   Очищает старые строки и добавляет новые (например, "2023–2024", "2024–2025").
    *   Позволяет массово продлить актуальность программ на несколько лет вперед.

### 3. Управление подписями и ФИО
*   **`run_signature_update.py`**: Умная замена ФИО в зонах подписей.
    *   Использует регулярные выражения для поиска (учитывает варианты "И.И. Иванов" и "Иванов И.И.").
    *   Меняет текст только в "зонах подписей" (где есть подчеркивания, даты или должности вроде "зав. кафедрой", "декан").
    *   Не затрагивает ФИО в основном тексте, где замена не требуется.

### 4. Вставка сканов (печать/подпись)
*   **`run_scan_insertion.py`**: Автоматически вставляет изображения (сканы) в нужные места документа.
    *   **Первая страница:** Вставка скана титульного листа.
    *   **Аннотация:** Поиск слова "АННОТАЦИЯ" и вставка скана после него.
    *   **Конец документа:** Вставка скана листа согласования.
    *   *Скрипт сам подбирает нужные сканы для каждого файла, ориентируясь на код дисциплины в названии файла.*

### 5. Извлечение аннотаций
*   **`run_annotation_extractor.py`**: Вырезает конкретную страницу из документа (по умолчанию 3-ю, где обычно находится аннотация) и сохраняет её в отдельный PDF-файл. Полезно для загрузки аннотаций в системы мониторинга.

### 6. Оптимизация и чистка
*   **`run_media_cleanup.py`**: "Облегчает" тяжелые файлы `.docx`.
    *   Находит и удаляет слишком крупные изображения и объекты, которые мешают загрузке файла в систему (например, если скан весит 20 МБ).
    *   Проводит "сборку мусора" внутри структуры файла, удаляя неиспользуемые медиа-ресурсы.
*   **`run_filename_cleanup.py`**: Приводит имена файлов в порядок: удаляет лишние пробелы, плюсы, точки и спецсимволы, делая их пригодными для хранения в архивах.

## 🛠 Технические требования

1.  **ОС**: Windows (так как используется `pywin32` для взаимодействия с MS Word).
2.  **Установленный Microsoft Word**.
3.  **Python 3.8+**.
4.  **Библиотеки**:
    *   `python-docx` — работа с текстом и таблицами.
    *   `pywin32` / `comtypes` — управление Word для конвертации.
    *   `PyMuPDF` (fitz) — обработка PDF и изображений.
    *   `pypdf` — извлечение страниц.
    *   `fpdf2` — генерация PDF из картинок.

---

## 📂 Структура проекта

*   **`core/`**: Базовый функционал редактирования (позиционирование плавающих изображений).
*   **`services/`**: Логика бизнес-процессов (обработка подписей, листов согласования, аннотаций).
*   **`utils/`**: Вспомогательные инструменты (конвертеры, чистильщики имен, поиск сканов).
*   **`run_*.py`**: Скрипты запуска. Пользователь взаимодействует именно с ними через консоль.

---

## 🚀 Инструкция по использованию

1.  **Подготовка**: Поместите все файлы РПД в отдельную папку. Если файлы в формате `.doc`, сначала запустите `run_doc_converter.py`.
2.  **Запуск**: Откройте терминал в папке проекта и запустите нужный скрипт:
    ```bash
    python run_approval_update.py
    ```
3.  **Ввод данных**: Скрипт спросит путь к папке и необходимые параметры (например, годы для листа согласования или ФИО для замены).
4.  **Результат**: Документы будут изменены "на месте" или сохранены в указанную вами папку.

> **Важно:** Всегда делайте резервную копию папки с документами перед массовой обработкой!

## 📝 Особенности работы со сканами (ScanFinder)
Для корректной работы автоматической вставки сканов (`run_scan_insertion.py`), файлы сканов должны называться по шаблону:
`[Код_дисциплины][Номер_страницы].[расширение]`
Например:
*   `Б1.О.01_1.jpg` (Титульный лист)
*   `Б1.О.01_2.jpg` (Аннотация)
*   `Б1.О.01_3.jpg` (Лист согласования)

Скрипт автоматически найдет их в папке со сканами и сопоставит с файлом РПД, в названии которого есть такой же код.
